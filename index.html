核心特性

为移动端 AI 工具链而生的完整协议与服务能力

完整 MCP 协议

严格遵循 JSON-RPC 2.0 over HTTP，内置 HTTP 服务器（支持 CORS），通过 JS Bridge + MutationObserver 自动监听工具调用。

DeepSeek 集成

DeepSeekChatBridge 单例跨 Activity 通信，每轮独立 requestId 并发管理；前端 500ms 轮询 + 120s 超时兜底，长对话不卡顿。

SSE 流式传输

started / chunk / status / done / plan / error 全事件流，HandlerThread 专线写入，HTTP chunked 编码确保完整交付。

上下文窗口哨兵

检测 LLM 因上下文被压缩而输出自然语言或漏回 id 时，自动重发完整系统提示词并弹窗提示，可持续无上限恢复。

内嵌 Python 3.14

JNI 嵌入 libpython3.14.so，无需 Termux。完整标准库 + asyncio/ssl/sqlite3，内置 ensurepip，可直接 pip 安装纯 Python 包。

Git 三层回退

内嵌 4.2MB 静态 git（0 依赖、含 HTTPS）→ 系统 git → dulwich 纯 Python 兜底，普通设备到 root 设备都能完整使用 Git。

Skill 热加载

支持知识型 + 工具型 skill 包，放入目录后 skills/reload 即生效；skill 工具自动注册，知识按需渐进式披露。

MT 管理器集成

自动连接 MT 管理器 APK MCP，注册 18+ 个 mt_apk_* 工具：APK 分析、资源修改、Smali 编辑、重新打包。

多线程性能优化

线程池管理 HTTP 连接，HandlerThread 专线处理 SSE 写入，提示词模板缓存 + 正则预编译，避免重复 I/O 与编译开销。

架构总览

MCP 服务端 · SSE 流 · DeepSeek 桥接 · 计划任务

协议接口

  • initialize — 初始化连接
  • tools/list — 获取全部工具
  • tools/call — 调用指定工具
  • skills/list — 已加载技能摘要
  • skills/reload — 热加载技能目录
  • notifications/initialized

SSE 事件流

  • started — 对话开始
  • chunk — LLM 逐字输出
  • status — 工具执行结果
  • done — 一轮回复完成
  • plan — 计划创建/更新/完成
  • context_compressed — 哨兵事件

计划 / 任务执行

  • LLM 输出 {"tasks":[...]}
  • PlanState 解析并创建任务
  • 逐个执行 + plan_update 推进
  • 依赖解析 / 优先级排序
  • 失败重试（最多 3 次）
  • 全部完成 → 自动生成总结

快速开始

编译安装 → 启动服务 → 接入 AI

1

编译安装

用 AIDE Pro 打开项目 → Build → Rebuild（先 Clean）；或用 Android Studio 执行 ./gradlew assembleDebug。

2

启动 MCP 服务

打开应用 →「启动 MCP 服务」，记录局域网地址（如 http://192.168.1.100:8080），浏览器访问即可打开 MCP 工具箱。

3

使用 DeepSeek 集成

启动服务后点击「打开 DeepSeek 助手」并登录，对话中 AI 会自动调用本地工具。

4

连接 MT 管理器

在 MT 管理器中启动 APK MCP，返回工具箱后 LLM 即可调用 mt_apk_* 工具。

5

安装技能

将 .md 技能文件放入运行时目录后调 skills/reload 即可生效。
